# Skill: Research

Guidelines for investigating problems and learning about the Go ecosystem.

## Sources

- **Official Go docs:** Use `go doc <package>` or reference pkg.go.dev
- **Go blog:** For language-level patterns and best practices
- **Web search:** Use when investigating specific errors or library usage

## Rules

- **Time-box:** Maximum 5 minutes per research topic. If no answer found, move on and log it.
- **Prefer official sources:** Go docs and stdlib reference over blog posts and Stack Overflow
- **Document findings:** If research reveals a broadly useful insight, add it to LEARNINGS.md
- **Verify claims:** Do not trust search results blindly. Test any code snippets before using them.

## When to Research

- Encountering an unfamiliar error message
- Needing to use a stdlib package you have not used before
- Investigating whether a dependency has a known bug
- Looking up Go idioms for a specific pattern

## When NOT to Research

- The answer is in the codebase (read the code first)
- The error message is self-explanatory
- You have already researched this topic in a previous session (check LEARNINGS.md)
